## Startup sequence

```
main()
  → Hive.initFlutter()
  → Hive.openBox<String>('candidates')   // candidate JSON cache
  → Hive.openBox<String>('statuses')     // local status overrides
  → configureDependencies()              // GetIt wiring via generated code
  → runApp(App())
```

`App` wraps a global `BlocProvider<CandidatesListCubit>` around `MaterialApp.router`.
`CandidateDetailCubit` is created on-demand per navigation push, not globally.

## Navigation

GoRouter with nested routes:

| Route | Screen |
| --- | --- |
| `/` | initial location — redirects to `/candidates` |
| `/candidates` | `CandidatesListScreen` |
| `/candidates/candidate/:id` | `CandidateDetailScreen` (nested) |

Route constants live in `AppRoutes`; router instance is `appRouter` exported from `core/router/router.dart`.
Error fallback shows a centered "Страница не найдена" scaffold.

## State management

Cubit + freezed sealed state classes. Each feature folder owns its cubit and state.

### `CandidatesListCubit`

Registered as `@injectable` (created fresh per `BlocProvider`).

**Status enum:** `initial → loading → loaded | error`

**State fields:**

| Field | Type | Purpose |
| --- | --- | --- |
| `status` | `CandidatesListStatus` | Loading phase |
| `allCandidates` | `List<Candidate>` | Full unfiltered list from repository |
| `filteredCandidates` | `List<Candidate>` | After search + verdict filter + sort |
| `displayedCandidates` | `List<Candidate>` | Current page slice (page × 10) |
| `searchQuery` | `String` | Active text search |
| `verdictFilter` | `String?` | Active verdict filter (`null` = all) |
| `sortBy` | `SortOption` | Active sort (default: `dateAdded`) |
| `currentPage` | `int` | 1-based page counter |
| `hasMore` | `bool` | Whether next page exists |
| `isOffline` | `bool` | Populated from repository |
| `errorMessage` | `String?` | Set on error status |

**Methods:**

- `load({forceRefresh})` — fetches from repository, subscribes to `candidatesStream` for live updates
- `search(query)` — re-filters in-memory, resets to page 1
- `filterByVerdict(verdict)` — re-filters in-memory, resets to page 1
- `sortBy(option)` — re-sorts in-memory, resets to page 1
- `loadNextPage()` — appends next 10 items to `displayedCandidates`

Pagination page size: **10**. Filtering and sorting are pure in-memory — no network call needed after initial load.

### `CandidateDetailCubit`

Registered as `@injectable`.

**Status enum:** `initial → loading → loaded | notFound | updatingStatus | error`

**State fields:** `status`, `candidate`, `errorMessage`

**Methods:**

- `load(id)` — calls `repository.getById(id)` (uses in-memory cache, no extra request)
- `updateStatus(newStatus)` — optimistic update: emits `updatingStatus` + updated candidate immediately, then calls repository; on error rolls back to previous candidate and emits `error` briefly before returning to `loaded`

## Data layer

### `Candidate` model

Freezed + json_serializable. Key fields:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | `String` | Unique identifier |
| `name` | `String` | |
| `position` | `String` | |
| `posLabel` | `String` | JSON: `pos_label` |
| `file` | `String` | CV link / path |
| `email`, `phone`, `city`, `tg` | `String` | Contact info |
| `exp` | `List<List<String>>` | Experience entries `[[title, period], …]` |
| `totalExp` | `String` | JSON: `total_exp` |
| `stack`, `edu` | `String` | |
| `verdict` | `String` | AI verdict text |
| `vc` | `String` | Verdict color key: `verdict-green / verdict-orange / verdict-red` |
| `criteria` | `List<List<String>>` | Criterion rows `[[label, status], …]` |
| `summary`, `questions` | `String / List<String>` | |
| `status` | `String` | Default `'new'`; one of `CandidateStatus` values |
| `dateAdded` | `String?` | JSON: `date_added` |

**`CandidateStatus` enum:** `newCandidate('new')`, `review('review')`, `invited('invited')`, `rejected('rejected')`.

### `CandidateRepository`

```dart
Future<List<Candidate>> getCandidates({bool forceRefresh = false});
Future<Candidate?> getById(String id);
Future<void> updateStatus(String id, String status);
Stream<List<Candidate>> get candidatesStream;
bool get isOffline;
```

**Offline-first strategy** in `CandidateRepositoryImpl`:

1. `getCandidates` — returns in-memory `_cache` if available and `!forceRefresh`
2. Checks connectivity via `connectivity_plus`
3. If online: fetches remote → overlays saved local statuses from Hive → writes back to Hive cache → emits on stream
4. If offline or remote throws: reads Hive cache, sets `_isOffline = true`
5. If no cache at all: throws (cubit shows error state)

`updateStatus` does an **optimistic update**: mutates `_cache` and emits on stream immediately, persists status to Hive, then fires the mock API call. On API error the cubit (not the repository) handles rollback.

### Datasources

| | Transport | Operations |
| --- | --- | --- |
| `RemoteDatasourceImpl` | Dio | `GET /candidates`, `PATCH /candidates/:id/status` |
| `LocalDatasourceImpl` | Hive | JSON blob in `candidates_box` (full list), status map in `statuses_box` |

Local keys: `candidates_list` (full JSON), `statuses_map` (id → status JSON map).

## Network (mock)

`MockInterceptor` replaces all network traffic:

| Request | Behaviour |
| --- | --- |
| `GET **/candidates` | Loads `mock/candidates.json` from assets, resolves after 300–800 ms simulated delay |
| `PATCH **/status` | Resolves 200 (~90%) or rejects with `DioExceptionType.badResponse` (~10%) — exercises error / rollback path |

Base URL configured in `RegisterModule`: `https://mock.api/`.

## Dependency injection

GetIt + Injectable. All wiring is code-generated into `injection.config.dart`.

| Class | Scope | Registered as |
| --- | --- | --- |
| `Dio` | `@lazySingleton` | `Dio` (via `@module RegisterModule`) |
| `Box<String>` candidates | `@lazySingleton` | `@Named('candidates_box')` |
| `Box<String>` statuses | `@lazySingleton` | `@Named('statuses_box')` |
| `RemoteDatasourceImpl` | `@LazySingleton` | `RemoteDatasource` |
| `LocalDatasourceImpl` | `@LazySingleton` | `LocalDatasource` |
| `CandidateRepositoryImpl` | `@LazySingleton` | `CandidateRepository` |
| `CandidatesListCubit` | `@injectable` | `CandidatesListCubit` |
| `CandidateDetailCubit` | `@injectable` | `CandidateDetailCubit` |

Entry point: `configureDependencies()` in `main()`.

## Utilities

### `candidate_filter.dart`

Pure function — no side effects, no context required.

```dart
List<Candidate> filterAndSort({
  required List<Candidate> candidates,
  required String query,
  required String? verdictFilter,
  required SortOption sortBy,
});
```

`SortOption` enum: `name` (alphabetical), `experience` (parses leading number from `totalExp`), `dateAdded` (string comparison — ISO date assumed).

### `verdict_colors.dart`

Maps `vc` string and criteria `status` string → `VerdictPalette` from `AppColorsExt`.

```dart
verdictPalette(context, vc)           // full VerdictPalette for 'verdict-green/orange/red'
verdictColor(context, vc)             // .dot
verdictContainerColor(context, vc)    // .background
onVerdictContainerColor(context, vc)  // .foreground
criteriaPalette(context, status)      // 'ok' / 'warn' / anything else
criteriaIcon(status)                  // check / warning / cancel icon
```

### `debouncer.dart`

Wraps a `Timer` for debouncing search input.
